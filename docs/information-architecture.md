# 정보 구조와 화면 방향

## 참고 방향

우아한형제들 기술블로그의 핵심은 화려한 랜딩 페이지가 아니라 콘텐츠 허브 구조다. 최신 글, 태그, 카테고리, 글 요약, 작성자 정보가 한눈에 들어오고, 독자는 관심 주제별로 글을 빠르게 찾을 수 있다.

이 블로그도 같은 방향을 따른다.

## 주요 페이지

### 홈

홈은 첫 화면부터 최신 글 목록을 보여준다.

필수 구성:

- 블로그 이름
- 짧은 소개 문장
- 주요 태그
- 최신 글 목록
- 추천 글 또는 대표 글

선택 구성:

- 최근 기술 일기
- 최근 프로젝트
- 외부 링크
- RSS 링크

### 글 상세

글 상세 페이지는 읽기 경험을 최우선으로 한다.

필수 구성:

- 제목
- 설명
- 작성일
- 태그
- 예상 읽기 시간
- 본문
- 이전/다음 글

선택 구성:

- 목차
- 관련 글
- 공유 버튼

### 태그 페이지

태그별 글 목록을 제공한다.

예상 태그:

- Frontend
- React
- Vite
- TypeScript
- CSS
- Web Performance
- Testing
- Architecture
- Retrospective

### Diary

날짜별 기술 일기를 보여준다. 블로그 글이 정리된 기술 아티클이라면, 일기는 하루 단위의 짧은 개발 기록이다.

목적:

- 오늘 배운 것
- 삽질한 문제와 해결 단서
- 읽은 글이나 문서
- 작업한 프로젝트의 흐름
- 다음에 다시 볼 작은 메모

목록 화면은 달력 중심으로 구성한다.

필수 구성:

- 월 단위 달력
- 일기가 있는 날짜 표시
- 선택한 날짜의 짧은 요약
- 이전/다음 달 이동
- 오늘로 이동

상세 화면은 날짜별 일기 하나를 보여준다.

필수 구성:

- 날짜
- 짧은 제목
- 오늘의 핵심 요약
- 본문
- 관련 태그
- 연결된 글 또는 프로젝트 링크

달력 UI 원칙:

- 날짜 셀은 작고 균일하게 유지한다.
- 일기가 있는 날은 점, 작은 바, 옅은 배경 중 하나로만 표시한다.
- 색상으로 감정을 과하게 표현하지 않는다.
- 모바일에서는 한 달 달력을 유지하되, 선택한 날짜의 미리보기는 달력 아래에 둔다.
- 빈 날짜도 레이아웃을 유지해 달력이 흔들리지 않게 한다.

### About

운영자 소개 페이지다.

포함할 내용:

### Projects

개인 프로젝트나 실험을 정리한다.

포함할 내용:

- 프로젝트 이름
- 설명
- 사용 기술
- GitHub 링크

- 관련 글 링크

## 내비게이션

상단 내비게이션은 단순하게 유지한다.

```txt
Home
Posts
Tags
Diary
Projects
About
```

모바일에서는 햄버거 메뉴보다 하단으로 접히는 간단한 메뉴를 우선 고려한다. 항목 수가 적기 때문이다.

## 디자인 원칙

- 본문 폭은 너무 넓히지 않는다.
- 카드는 필요한 곳에만 사용한다.
- 글 목록은 제목, 날짜, 요약, 태그 중심으로 구성한다.
- 일기는 달력, 날짜, 짧은 요약 중심으로 구성한다.
- 태그는 탐색 도구로 쓰되 과하게 장식하지 않는다.
- 색상은 흰 배경, 검정 텍스트, 하나의 포인트 컬러를 기본으로 한다.
- 코드 블록, 인용문, 표, 이미지 캡션 스타일을 초기에 정한다.
